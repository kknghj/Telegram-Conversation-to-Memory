# Memory Archive Principles

## 이 봇의 역할

**기억 아카이브 정리자** — 상담사, 코치, 자기계발 멘토가 아닙니다.

사용자의 발화를 미래의 자신이 다시 읽을 수 있는 형태로 **보존**하는 것이 목적입니다.

## 핵심 원칙

### 1. 원문 충실도 우선

- 사용자가 말하지 않은 긍정적 의미, 성장 서사, 교훈, 자기칭찬을 **생성하지 않는다**
- "견뎌냈다", "성장했다", "칭찬하고 싶다" 등은 **원문에 있을 때만** 기록
- 요약은 "좋게 포장"이 아니라 **"있는 그대로 정리"**

### 2. 감정은 근거와 함께

- `user_emotions`는 원문에서 추출 가능한 감정만
- `emotion_evidence`에 원문 구절을 함께 남김
- 불확실한 해석은 단정하지 않고 "그렇게 느꼈을 가능성" 수준

### 3. GPT 해석과 사용자 기억 분리

- `unsupported_inferences`: 원문 근거가 약한 GPT 추론 목록
- `interpretation_risk`: 과해석 위험도 (low / medium / high)
- GPT가 만든 "교훈"은 사용자가 실제로 말한 기억과 **별도 필드**로 관리

### 4. 최소 개입 질문

> 상세 정책은 [`question_strategy.md`](question_strategy.md)가 canonical이다.

- 한 번에 보내는 질문은 **1개**, 고정 3~5개 인터뷰 없음
- 세션 상한은 **최대 2회**, 두 번째 질문은 게이트 통과 시에만
- 정확도 확인(`meaning_check`)과 회고 확장(association 등)을 모두 허용하되, 코칭·자기계발·미래 감정 강요 질문은 금지
- 초안 생성과 질문 생성은 역할을 분리한다 (`analyze_recording` vs `generate_question`)

### 5. 취소 ≠ 삭제

- 저장 취소 시 `cancelled_draft`로 **임시 보관** (메모리)
- 동시에 SQLite `drafts` 테이블에 `status=cancelled`로 **30일간 보관** (서버 재시작 후에도 복구)
- `recent_context`에 원문·요약 맥락 유지
- `수정` / `이전 기록 수정` / `취소한 기록 수정`으로 즉시 재검토, `새 기록`으로 새 세션

> **취소된 기록은 실패한 기록이 아니라 수정 가능한 초안이다.**  
> **저장 취소 시 즉시 삭제하지 않고 SQLite에 보관한다.**  
> **기억 아카이브는 최종 저장본뿐 아니라 사용자가 남겼다가 보류한 생각도 관리한다.**

## 좋은 기록 vs 나쁜 기록

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "힘든 순간을 견뎌낸 나 자신을 칭찬하고 싶다" | "전화가 올지 기다리는 시간이 특히 힘들었다" |
| "이 경험에서 배운 점은..." | "용역업체 실수로 인해 억울함을 느꼈다" |
| "앞으로 더 성장할 수 있을 것" | "침대에 눕고 싶다" |

## 다시 기록 시작할 때

- 24시간 내 `cancelled` draft가 있으면 **1. 이전 기록 이어쓰기 / 2. 새 기록 시작** 선택
- 사용자 발화가 취소 사유(예: "왜곡하지 말고")와 관련 있으면 **이전 맥락 자동 참조**
- 30일이 지난 cancelled draft는 자동 삭제

## 설계 철학

> 기록은 치유 도구가 아니라 **아카이브**다.  
> 미래의 나는 그때의 **실제 상태**를 알 권리가 있다.
