# User Conversation Profile

이 문서는 Conversation-to-Memory의 **대상 사용자**가 어떤 방식으로 대화할 때 기록 품질이 높아지는지를 정의한다.  
회고형 대화 에이전트가 질문을 고를 때, 상담사·코치 톤이 아니라 **이 사용자의 사고 습관**에 맞추기 위한 기준이다.

> 제품 원칙(원문 충실, 사용자 승인)은 `vision.md`, `MEMORY_ARCHIVE_PRINCIPLES.md`를 따른다.  
> 이 문서는 그 원칙 안에서 **어떤 질문이 이 사용자에게 맞는가**를 다룬다.

---

## 1. 사용자 전제

### 1.1 핵심 성향

이 사용자는 **항상 내면을 파고드는 타입이 아니다.**

- 감각과 호기심이 살아날 때 관찰력이 급격히 살아난다.
- “왜 그렇게 느꼈나요?”만 반복하면 대화가 닫히기 쉽다.
- 구체적인 장면, 사물, 소리, 분위기, 우연한 연상이 나오면 말이 풀린다.
- 독서모임처럼 **꼬리에 꼬리를 무는 연상형 대화**를 선호한다.

### 1.2 대화가 잘 열리는 조건

| 조건 | 설명 | 에이전트 대응 |
|------|------|----------------|
| 감각적 단서 | 장소, 소리, 빛, 온도, 물건, 몸의 반응 | 관찰 포인트를 질문에 반영 |
| 호기심 트리거 | “이건 뭔가 비슷한데”, “왜 이런 게 떠오르지” | 연상·대비·비유로 확장 |
| 외부 사건 앵커 | 실제 있었던 일, 사람, 대화, 실수 | 사건 중심으로 먼저 고정 |
| 느슨한 연결 | 주제가 살짝 옆으로 새는 것을 허용 | 한 가지 주제만 강제하지 않음 |

### 1.3 대화가 닫히는 조건

| 조건 | 설명 | 에이전트 대응 |
|------|------|----------------|
| 의미 확인 반복 | “이렇게 저장해도 될까요?”만 연속 | `meaning_check` 빈도 제한 |
| 내면 강요 | 감정 명명·가치 판단을 먼저 요구 | 감정은 근거가 있을 때만 |
| 성장 서사 유도 | 배운 점, 견뎌낸 자신, 교훈 | 금지 (기존 fidelity 정책 유지) |
| 피로·중단 신호 | 짧은 답, “됐어”, “모르겠어” | 질문 중단, 요약 단계로 |

---

## 2. 선호하는 질문 방식

### 2.1 연상형 확장 (association)

- 한 발화에서 떠오른 **단어·장면·인물**을 다음 질문의 출발점으로 삼는다.
- “그때 떠올랐던 다른 장면이 있나요?”
- “그 표현이 다른 기억의 무엇과 닮았나요?”

### 2.2 대비와 각도 전환 (contrast)

- 같은 사건을 다른 시점·다른 사람 시선·다른 맥락에서 본다.
- “그날과 오늘, 그 상황을 다르게 느끼는 지점이 있나요?”

### 2.3 기억 연결 (memory_link)

- 최근 저장된 기억·이번 대화 안의 반복 패턴을 **사실 연결**로만 제시한다.
- “지난번 ○○ 이야기에서도 비슷한 말을 했는데, 이번엔 무엇이 달랐나요?”
- 해석을 덧붙이지 않고, 사용자가 연결 여부를 판단하게 한다.

### 2.4 미해결 지점 열기 (unresolved_point)

- 사용자가 말하다 멈춘 곳, 모순처럼 보이는 곳, 스스로 “모르겠다”고 한 곳.
- “방금 ‘이상하게’라고 했는데, 그게 어떤 느낌에 가까웠나요?” — 단, 감정 단정 금지.

### 2.5 가치·선호 탐색 (value_probe) — 제한적 사용

- 사용자가 이미 드러낸 선택·불편·선호를 **사실 확인** 수준으로만 다룬다.
- “그때 중요하게 여긴 건 시간이었나요, 공정함이었나요?” — 사용자가 말한 단서 기반.

### 2.6 미래 회고 각도 (future_reflection) — 매우 제한적

- “나중에 다시 읽었을 때 무엇이 남으면 좋겠나요?”처럼 **기록 의도**만 묻는다.
- 미래 감정 예측, 조언 요청, 행동 계획 유도는 하지 않는다.

---

## 3. 피해야 할 질문 방식

### 3.1 과도한 meaning_check

- 저장 문구 확인은 필요할 때만 1회.
- 연속 세션에서도 직전 질문이 `meaning_check`였다면 다음은 다른 유형을 우선한다.

### 3.2 코칭·상담 톤

- “어떻게 극복하셨나요?”
- “그 경험에서 배운 점은?”
- “스스로에게 해주고 싶은 말은?”

### 3.3 감정 라벨 강요 (emotion_probe 남용)

- 원문에 감정 단서가 없는데 “화가 났나요, 슬펐나요?”로 좁히기.
- 감정은 `emotion_evidence`가 생길 때까지 관찰·연상 질문으로 접근.

### 3.4 기록 방향 선점

- 에이전트가 “핵심은 ○○입니다”라고 정한 뒤 동의만 구하는 패턴.
- 항상 사용자 표현을 먼저 인용하고, 해석은 `model_interpretation`으로 분리해 표시.

## 4. 기록 기준

### 4.1 무엇을 장기 기억 후보로 볼 것인가

| 우선순위 | 내용 | 이유 |
|----------|------|------|
| 높음 | 구체적 사건 + 당시 감각·맥락 | 미래의 나가 상황을 복원할 수 있음 |
| 높음 | 사용자 고유 표현 (`key_phrases`) | 왜곡 없이 목소리가 남음 |
| 중간 | 반복되는 주제·관계·프로젝트 (`emerging_themes`) | 회고 시 패턴 인식 |
| 중간 | 아직 닫히지 않은 생각 (`open_questions`) | 다음 대화의 씨앗 |
| 낮음 (단독 저장 비권장) | 에이전트 해석·추론 | `user_confirmed` 전에는 본문에 섞지 않음 |

### 4.2 기억 유형 (memory_type) 가이드

| 유형 | 설명 | 예 |
|------|------|-----|
| `event` | 특정 시점의 사건 | 회의, 갈등, 실수 |
| `observation` | 관찰·감각 중심 | 장면, 분위기, 디테일 |
| `relation` | 사람·관계 맥락 | 팀장, 동료, 가족 |
| `pattern` | 반복되는 경향 | 비슷한 불편, 반복 주제 |
| `reflection_seed` | 아직 정리되지 않은 생각 | 열린 질문, 미완성 연상 |

한 세션에 여러 유형이 섞일 수 있다. **하나만 고르게 강제하지 않는다.**

### 4.3 reflection_value (회고 가치) 판단

| 등급 | 기준 |
|------|------|
| `high` | 사건·감각·고유 표현이 풍부하고, 1~2주 뒤 다시 읽을 이유가 분명함 |
| `medium` | 맥락은 있으나 아직 얇음; 연상 질문 1~2회로 보강 가능 |
| `low` | 사실 확인만 필요하거나, 사용자가 마무리 의사를 밝힘 |

`reflection_value`가 `low`여도 저장을 막지 않는다. **질문을 더 할지 말지**의 신호로 쓴다.

### 4.4 user_confirmed 원칙

- `memory_candidate`에 들어가는 **해석·강조·구조화**는 사용자 검토 후 확정.
- `model_interpretation`은 “에이전트가 이렇게 읽었다”는 메타이며, 사용자가 수정·거부할 수 있다.
- 승인 전 JSON에는 후보와 해석을 구분해 보여준다.

## 5. 세션 내 대화 리듬

### 5.1 권장 흐름

```text
자유 기록 (RECORDING)
  → (선택) 회고형 질문 1~N회 (FOLLOWUP / REFLECTION)
  → 요약·JSON 검토 (REVIEW)
  → 저장 / 수정 / 취소
```

현재 MVP는 `요약` 트리거 후 질문 0~1회.  
회고형 에이전트로 확장할 때는 **기록 중에도** 짧은 연상 질문을 허용할 수 있으나, MVP에서는 `요약` 이후 단계에 집중한다.

### 5.2 한 세션 질문 수 가이드

| 단계 | 권장 |
|------|------|
| 현재 (프로토타입) | 최대 1회 (`meaning_check` 위주) |
| 회고형 MVP | 2~4회, 유형 다양화 필수 |
| 성숙 단계 | 사용자 설정·피로 신호에 따라 가변 |

### 5.3 질문 간 간격

- 질문마다 **서로 다른 `question_mode`**를 우선한다.
- 같은 모드 2연속 금지 (`meaning_check`는 세션당 최대 1회 권장).
