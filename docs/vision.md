# Vision - Conversation-to-Memory

## 제품 정의

Conversation-to-Memory는 사용자가 Telegram에 남긴 이야기를 미래의 자신이 다시 읽을 수 있는 형태로 보존하는 **개인 기억 아카이브**입니다.

AI의 역할은 사용자를 상담하거나 더 나은 이야기로 바꾸는 것이 아닙니다. 사용자가 실제로 말한 사건, 감정, 맥락을 잃지 않도록 정리하고, 저장 여부를 사용자에게 돌려주는 것입니다.

## 핵심 가치

> 미래의 나는 그때의 실제 상태를 알 권리가 있다.

기록의 품질은 얼마나 긍정적이거나 교훈적인지가 아니라 다음 기준으로 판단합니다.

1. 원문에 충실한가
2. 사용자가 말하지 않은 의미를 덧붙이지 않았는가
3. 감정과 해석의 근거가 드러나는가
4. 사용자가 저장 전에 내용을 통제할 수 있는가
5. 시간이 지난 뒤 당시 상황을 복원할 수 있는가

## 왜 만드는가

- 일기는 빈 화면에서 시작해야 하므로 쓰기 부담이 큽니다.
- 단순 메모는 남기기 쉽지만 사건과 감정의 맥락이 사라지기 쉽습니다.
- 생성형 AI는 기록을 매끄럽게 만들 수 있지만, 사용자의 경험을 긍정적으로 왜곡하거나 교훈으로 바꿀 위험이 있습니다.
- 대화의 편의성과 아카이브의 정확성을 결합하면 기록 마찰을 낮추면서 기억의 신뢰도를 지킬 수 있습니다.

## 해결 방식

| 문제 | 제품 접근 |
|------|-----------|
| 긴 글을 한 번에 쓰기 어렵다 | Telegram에서 여러 메시지로 자유롭게 기록 |
| AI가 경험을 좋게 포장한다 | 원문 충실도 프롬프트와 코드 후처리 병행 |
| 감정 해석이 과해질 수 있다 | 감정 근거와 해석 위험도를 구조화 |
| 질문이 기록을 다른 방향으로 이끈다 | 정확도 확인 질문만 최대 1개 |
| 실수로 저장될 수 있다 | 요약·JSON 검토 후 명시적 승인 |
| 취소하면 생각이 사라진다 | 취소 초안을 일정 기간 보관하고 복구 |
| 시간이 지나면 찾기 어렵다 | 주제, 감정, 사람, 프로젝트, 태그로 구조화 |

## 제품 경계

이 제품은 다음 역할을 하지 않습니다.

- 상담사, 코치, 자기계발 멘토
- 사용자의 경험에서 교훈을 만들어내는 도구
- 일정, 할 일, 업무를 대신 관리하는 범용 비서
- 사용자 승인 없이 기억을 수집하는 자동 기록기
- 현재 단계에서의 다중 사용자 협업 서비스

## 성공 기준

### 기능적 성공

- 사용자가 `기록 시작`부터 승인·저장까지 중단 없이 완료할 수 있다.
- 저장된 JSON만 읽어도 당시 사건과 감정의 근거를 이해할 수 있다.
- 수정과 취소가 실제 저장 전에 정상적으로 작동한다.
- 취소한 초안을 서버 재시작 후에도 복구할 수 있다.

### 제품적 성공

- 저장된 기록을 1~2주 뒤 읽었을 때 “내가 실제로 했던 말과 상태에 가깝다”고 느낀다.
- AI가 추가한 과도한 긍정 해석이나 성장 서사를 사용자가 거의 발견하지 않는다.
- 질문이 부담스럽거나 기록 방향을 유도한다고 느끼지 않는다.
- 자유 메모보다 다시 읽고 싶은 기록을 더 쉽게 남길 수 있다.

## 현재 검증 질문

기능 추가보다 먼저 다음을 확인합니다.

1. 원문 충실도가 사용자의 저장 신뢰를 높이는가
2. 후속 질문 0~1개가 기록 부담을 낮추는가
3. 구조화 필드가 실제 회고 시간을 줄이는가
4. 취소 초안 복구가 기록 포기를 수정 행동으로 바꾸는가
5. 30~50개의 기억이 쌓인 뒤 검색·주간 회고가 정말 필요한가
