# Vision - Memory-Based Personal AI Agent

## 제품 정의

Conversation-to-Memory의 현재 구현은 Telegram에 남긴 이야기를 원문에 충실한 구조화 기억으로 저장하는 개인 기억 아카이브입니다.

하지만 이 프로젝트의 장기 목표는 단순한 일기장, 메모 저장소, 회고 도구가 아닙니다. 궁극적으로는 사용자의 장기 기억을 바탕으로 흥미로운 콘텐츠를 지속적으로 발견하고 발행하는 **기억 기반 개인 AI 에이전트**를 만드는 것입니다.

이 에이전트는 사용자가 매번 회고를 요청하기를 기다리지 않습니다. 축적된 기록을 읽고, 느슨한 연결을 발견하고, 아직 덜 익은 생각을 보관하다가, 충분히 읽을 만한 콘텐츠가 되었을 때 사용자에게 조용히 알려줍니다.

> 새로운 콘텐츠가 준비되었습니다.

사용자는 그 알림을 보고 읽을지, 넘길지, 수정할지, 피드백을 줄지 선택합니다.

## 북극성

> 삶을 더 효율적으로 관리하는 것이 아니라, 삶을 더 흥미롭게 읽어주는 개인 AI.

이 제품은 생산성 도구나 코칭 도구가 아닙니다. 할 일을 정리하거나 더 나은 사람이 되라고 말하는 존재도 아닙니다. 목표는 사용자의 삶을 소재로 삼아, 시간이 지나야 보이는 연결과 장면과 반복을 읽어내는 것입니다.

좋은 결과물은 사용자가 이렇게 느끼게 해야 합니다.

- "이걸 내가 직접 요청하지 않았는데도 읽고 싶다."
- "내 기록을 멋대로 미화하지 않았는데도 새롭게 보인다."
- "이건 조언이 아니라, 내 삶을 다른 각도에서 읽은 글이다."
- "당장 쓸모가 있어서가 아니라 흥미로워서 다시 열어보게 된다."

## 현재 구현과 장기 비전의 구분

### 현재 구현

현재 시스템은 다음을 수행합니다.

- Telegram에서 사용자의 자유 기록을 받습니다.
- 원문 기반으로 기억 초안 JSON을 생성합니다.
- 감정, 사람, 프로젝트, 태그, 핵심 표현, 열린 질문, 회고 씨앗을 구조화합니다.
- 과해석, 성장 서사, 긍정적 재해석을 탐지하고 억제합니다.
- 사용자가 검토하고 `저장`한 기억만 최종 저장소에 보관합니다.
- 취소된 초안도 일정 기간 복구 가능한 상태로 보관합니다.

즉 현재 제품은 **신뢰할 수 있는 기억 기질(memory substrate)** 을 만드는 단계입니다.

### 장기 비전

장기적으로 시스템은 다음 능력을 갖습니다.

- 기억을 단순히 저장하지 않고, 시간이 지나며 다시 읽습니다.
- 여러 기억 사이의 반복 표현, 가치관, 장면, 관계, 프로젝트 흐름을 연결합니다.
- 콘텐츠 후보를 내부적으로 만들고, 근거가 약하면 보류합니다.
- 초안을 즉시 발행하지 않고, 추가 기억과 반례가 쌓일 때까지 숙성시킵니다.
- 발행 시점과 빈도를 관리해 사용자 경험을 해치지 않습니다.
- 사용자의 읽기, 저장, 무시, 수정, 피드백을 다음 콘텐츠 품질에 반영합니다.

## 핵심 경험

사용자는 "회고해줘"라고 요청하지 않아도 됩니다.

1. 사용자는 평소처럼 Telegram에 기록을 남깁니다.
2. 시스템은 승인된 기억과 보류된 회고 씨앗을 축적합니다.
3. 내부 에이전트들이 기억에서 흥미로운 연결 후보를 발견합니다.
4. 근거가 충분하지 않은 후보는 초안 상태로 남아 계속 수정됩니다.
5. 읽을 만한 밀도와 타이밍이 갖춰지면 알림을 보냅니다.
6. 사용자는 콘텐츠를 읽고, 넘기고, 수정하거나, 피드백을 남깁니다.
7. 그 반응은 이후 발행 판단과 해석 품질에 반영됩니다.

중요한 점은 콘텐츠가 생성 즉시 사용자에게 노출되지 않는다는 것입니다. 이 제품은 빠르게 많이 생성하는 시스템이 아니라, 기억을 오래 들고 있다가 적절한 순간에 꺼내는 시스템입니다.

## 역할 기반 시스템 철학

이 프로젝트는 단일 AI가 모든 일을 처리하는 구조로 발전하지 않습니다. 좋은 콘텐츠 경험을 위해 역할과 책임을 분리합니다.

| 역할 | 핵심 책임 |
|------|-----------|
| Archivist | 사용자의 원문을 보존하고, 기억을 신뢰 가능한 구조로 정리한다. |
| Reporter | 기억 속에서 흥미로운 연결점, 반복, 장면, 콘텐츠 후보를 많이 발견한다. |
| Editor | 후보의 근거를 검토하고, 과해석과 반례를 찾고, 더 좋은 해석과 표현으로 다듬는다. |
| Editor-in-Chief | 무엇을 언제 사용자에게 보여줄지 결정하고, 알림 빈도와 읽기 경험을 관리한다. |
| Feedback Steward | 발행 후 반응을 해석해 다음 콘텐츠의 기준과 선호를 갱신한다. |

역할 이름은 구현 세부사항이 아니라 제품 책임을 설명하기 위한 개념입니다. 실제 코드에서는 서비스, 큐, 배치 작업, 에이전트 호출, 상태 머신 등으로 나뉠 수 있지만 문서와 설계는 이 책임 분리를 유지해야 합니다.

## 콘텐츠의 성격

이 프로젝트가 만들 콘텐츠는 다음에 가깝습니다.

- 어떤 표현이 여러 시기에 반복되는지 보여주는 짧은 글
- 과거의 장면과 최근의 판단 기준을 연결하는 메모
- 아직 답이 없지만 계속 돌아오는 질문 모음
- 특정 프로젝트나 관계를 둘러싼 관점의 변화
- 사용자의 말투와 감각을 살린 작은 에세이
- "그때의 나"와 "지금의 나"가 다르게 읽는 지점

반대로 다음은 목표가 아닙니다.

- 생산성 리포트
- 습관 개선 코칭
- 심리 상담
- 자기계발식 교훈 생성
- 사용자를 긍정적으로 포장하는 감성 일기
- 매주 자동으로 찍어내는 요약 보고서

## 제품 원칙

### 1. 원문은 언제나 최상위 근거다

기억 기반 콘텐츠가 아무리 흥미로워도, 원문에 없는 사실을 만들어내면 제품의 신뢰가 무너집니다. 모든 해석은 원문, 승인된 기억, 사용자 피드백으로 돌아갈 수 있어야 합니다.

### 2. 해석은 사실과 분리한다

AI가 발견한 연결과 관점은 가치가 있지만, 그것은 사용자 발화 자체가 아닙니다. 콘텐츠는 해석을 제공할 수 있으나 근거와 불확실성을 함께 드러내야 합니다.

### 3. 콘텐츠는 숙성된다

한 번의 기록에서 바로 좋은 글이 나오지 않을 수 있습니다. 어떤 후보는 몇 주 동안 보류되고, 다른 기억과 만나며 더 나아질 수 있습니다. 발행하지 않는 결정도 중요한 제품 판단입니다.

### 4. 사용자는 요청자가 아니라 독자다

사용자는 매번 프롬프트를 잘 써야 하는 사람이 아닙니다. 이 시스템은 사용자가 남긴 기록을 바탕으로, 사용자가 읽을 만한 것을 먼저 준비하는 방향으로 발전해야 합니다.

### 5. 알림은 콘텐츠 경험의 일부다

좋은 콘텐츠도 너무 자주, 너무 이르게, 너무 확신에 차서 오면 부담이 됩니다. Editor-in-Chief 역할은 콘텐츠 품질뿐 아니라 타이밍, 빈도, 피로도, 사용자의 읽기 리듬을 관리합니다.

### 6. 피드백은 취향을 배우는 수단이다

사용자가 어떤 콘텐츠를 읽고, 넘기고, 저장하고, 불편해했는지는 장기적으로 중요합니다. 단, 피드백은 사용자를 조작하기 위한 최적화가 아니라 더 흥미롭고 덜 왜곡된 읽기를 위한 기준입니다.

## 성공 기준

### 현재 단계의 성공

- 사용자가 기록 시작부터 승인 저장까지 안정적으로 완료할 수 있다.
- 저장된 기억이 원문과 감정 근거를 충실히 보존한다.
- `reflection_seed`, `key_phrases`, `open_questions` 같은 필드가 장기 콘텐츠 후보의 재료가 된다.
- 사용자가 1-2주 뒤 다시 읽었을 때 "내 말에 가깝다"고 느낀다.

### 다음 단계의 성공

- 시스템이 기억 사이의 연결 후보를 만들 수 있다.
- 후보마다 근거, 반례, 해석 위험도를 함께 관리할 수 있다.
- 사용자가 요청하지 않아도 내부 초안이 쌓인다.
- 충분히 익지 않은 콘텐츠를 발행하지 않는 판단이 가능하다.

### 장기 성공

- 사용자가 알림을 받았을 때 부담보다 호기심을 먼저 느낀다.
- 발행된 콘텐츠가 조언 없이도 새 관점을 제공한다.
- 사용자는 AI가 자신을 미화하거나 단정한다고 느끼지 않는다.
- 시간이 지날수록 콘텐츠가 더 개인적이고, 더 근거 있고, 더 읽을 만해진다.

## 제품 경계

이 제품은 다음 역할을 하지 않습니다.

- 상담사, 코치, 자기계발 멘토
- 일정, 할 일, 업무를 대신 관리하는 범용 비서
- 사용자 승인 없이 민감한 기억을 수집하는 자동 기록기
- 원문 근거 없이 사용자의 성격이나 심리를 단정하는 분석기
- 단기 참여율을 위해 과장된 알림을 보내는 콘텐츠 앱

다만 이 제품은 단순한 저장소에 머물지도 않습니다. 저장은 목적지가 아니라, 나중에 흥미로운 콘텐츠를 만들기 위한 신뢰 기반입니다.
