# MVP Scope

## MVP 목표

이 MVP는 “AI와 대화하면 더 좋은 일기를 쓸 수 있는가”가 아니라 다음 가설을 검증합니다.

> 사용자의 말을 왜곡하지 않고 구조화하며, 저장 통제권을 사용자에게 남기는 흐름이 다시 읽을 가치가 있는 기억을 만드는가?

## 포함 범위

### 채널과 인프라

- Telegram Bot (`python-telegram-bot`)
- OpenAI API를 이용한 원문 분석과 JSON 구조화
- `.env` 기반 비밀값 관리
- 승인된 기억의 Local JSON 저장
- 진행·취소 초안의 SQLite 영속화

### 사용자 흐름

1. `기록 시작`
2. 여러 메시지로 자유 기록
3. `요약`
4. 원문 기반 기억 초안 생성
5. 필요한 경우 정확도 확인 질문 최대 1개
6. 요약과 JSON 검토
7. `저장`, `수정`, `취소`
8. 취소 초안 복구 또는 새 기록 시작

### 품질 보호

- 원문에 없는 성장 서사와 긍정적 재해석 금지
- 감정별 원문 근거 기록
- 근거가 약한 추론 목록화
- 해석 위험도 `low`, `medium`, `high` 표시
- 후속 질문 1회 제한

### 저장 스키마

```json
{
  "timestamp": "...",
  "topic": "...",
  "event_summary": "...",
  "user_emotions": [],
  "emotion_evidence": [],
  "people": [],
  "projects": [],
  "tags": [],
  "memory_candidate": "...",
  "interpretation_risk": "low",
  "unsupported_inferences": [],
  "needs_followup": false,
  "followup_question": "",
  "conversation": [],
  "approved": true
}
```

승인된 기억의 파일명은 `data/memories/YYYY-MM-DD_HHMMSS.json`입니다.

### 초안 보관

| 상태 | 정책 |
|------|------|
| `active` | 입력 중 갱신, 7일 방치 시 `cancelled` 전환 |
| `cancelled` | 복구 가능, 30일 후 삭제 |
| `saved` | 삭제하지 않음 |

## 제외 범위

- 상담, 코칭, 자기계발 조언
- 고정된 다문항 인터뷰
- 사용자 승인 없는 자동 저장
- Supabase와 다중 사용자 인증
- Vector DB와 의미 검색
- 자동 주간·월간 회고
- Notion·Hermes 연동
- 웹 UI
- 음성·사진 입력

## 실험 가설

1. **H1 원문 신뢰**: 사용자는 긍정적 재해석보다 원문에 충실한 요약을 더 신뢰한다.
2. **H2 최소 개입**: 후속 질문을 0~1개로 제한하면 기록 완료 부담이 낮아진다.
3. **H3 승인 통제**: 저장 전 검토와 수정 기능이 의도치 않은 기억 축적을 막는다.
4. **H4 구조화 회고**: 사건, 감정 근거, 사람, 프로젝트, 태그가 나중의 맥락 복원에 도움을 준다.
5. **H5 취소 복구**: 취소 초안을 보관하면 기록 포기가 영구적인 내용 손실로 이어지지 않는다.
6. **H6 채널 적합성**: Telegram만으로 개인 기록 루틴을 유지할 수 있다.

## 검증 항목

| 항목 | 방법 | 초기 통과 기준 |
|------|------|----------------|
| 흐름 완주 | 실제 기록 세션 반복 | 5회 연속 저장 완료 |
| 원문 충실도 | 저장 전 원문과 요약 비교 | 5건 중 4건 이상 수정 없이 승인 |
| 과해석 억제 | 성장 서사·교훈 생성 여부 확인 | 사용자 미발화 추론 0건 목표 |
| 질문 부담 | 후속 질문 횟수와 중단 여부 기록 | 기록당 최대 1개, 질문 때문에 중단 0건 |
| 승인 UX | 수정·취소·저장 시나리오 실행 | 의도치 않은 저장 0건 |
| 초안 복구 | 재시작 후 취소 초안 복구 | 주요 복구 시나리오 100% 성공 |
| 회고 가치 | 1~2주 뒤 다시 읽기 | 5점 척도 평균 4점 이상 |
| 운영 안정성 | 1주 이상 개인 사용 | 데이터 손실 없이 동작 |

## 다음 단계 진입 조건

다음 인프라 확장은 아래 조건을 충족한 뒤 결정합니다.

- 최신 스키마로 승인된 기억 30~50개 축적
- 1~2주 후 회고 가치 확인
- 반복적으로 필요한 검색 질문 발견
- 로컬 저장의 백업·멀티디바이스 한계가 실제 문제로 나타남
