대화 전체를 분석해 "기억 후보"를 구조화된 JSON으로 추출하세요.

## 출력 규칙
- 반드시 유효한 JSON만 출력 (마크다운·설명 없음)
- 한국어로 작성

## JSON 스키마
{
  "topic": "핵심 주제 (한 문장)",
  "emotion": "주요 감정 (1~3단어)",
  "people": ["관련된 사람 이름 또는 역할"],
  "projects": ["관련 프로젝트·일·활동"],
  "tags": ["검색·회고용 태그 2~5개"],
  "summary": "3~5문장 요약 (회고 시 읽기 좋게)",
  "memory_candidate": "나중에 다시 읽을 가치가 있는 핵심 기억 (2~4문장, 1인칭 또는 서사체)"
}

## 원칙
- 대화에 없는 내용을 지어내지 마세요
- people/projects가 없으면 빈 배열 []
- memory_candidate는 단순 요약이 아니라 "미래의 나"에게 남기는 기억 조각
